# SVC 插件快速使用指南 - 让 AI 访问远程文件

## 🎯 核心功能

这个插件让你的 **AI 智能体能够直接访问和修改远程服务器上的文件**，同时支持本地编辑自动同步。

## 📦 安装

```bash
# 在插件目录下执行
code --install-extension svc-extension-0.1.0.vsix
```

## 🚀 5 分钟上手

### 步骤 1: 添加服务器

1. 按 `F5` 启动插件开发环境
2. 在新窗口中，点击左侧 **SVC 服务器** 图标
3. 点击 `+` 添加服务器
4. 填写信息：
   ```
   服务器名称: 训练服务器
   主机地址: 你的服务器IP
   端口: 22
   用户名: root (或你的用户名)
   认证: 密码 或 私钥
   远程路径: /root (会自动设置)
   ```

### 步骤 2: 连接到服务器

1. 点击服务器列表中的服务器
2. 在弹出的文件夹浏览器中选择你要访问的文件夹
   例如: `/root/PALS-safe-Jun-21th-CIFAR100/topology_daes`
3. 等待扫描完成（可以随时点击"取消"）

### 步骤 3: 让 AI 访问文件

现在 AI 智能体可以直接访问文件了！

#### 示例 1: 让 AI 分析代码

```
你: "请分析 train.py 中的 main 函数"
AI: [读取缓存中的 train.py 文件]
AI: "main 函数做了以下几件事..."
```

#### 示例 2: 让 AI 修改代码

```
你: "在 model.py 的第 50 行添加一个 print 语句"
AI: [读取文件] → [修改] → [保存]
插件: 自动上传到服务器 ✅
```

#### 示例 3: 让 AI 创建新文件

```
你: "创建一个新的 config.yaml 文件，包含训练参数"
AI: [在缓存目录创建文件]
插件: 自动上传到服务器 ✅
```

## 🔥 工作原理

```
1. 连接服务器 → 扫描目录结构
   └─ 只同步文件夹和文件名，不下载内容

2. 你或 AI 打开文件
   └─ 插件自动下载文件内容到本地缓存

3. 编辑并保存文件
   └─ 插件自动上传到远程服务器

4. AI 可以直接访问
   └─ 因为文件在本地缓存目录中
```

## 📁 缓存位置

文件缓存在：
- **Windows**: `C:\Users\你的用户名\.svc-cache\`
- **Linux/Mac**: `~/.svc-cache/`

你可以直接在这里看到所有下载的文件。

## 🎯 .svcignore - 忽略大文件

在远程文件夹创建 `.svcignore` 来忽略不需要的文件：

```gitignore
# 忽略模型文件
*.pkl
*.pth
*.pt
*.ckpt

# 忽略数据集
data/
dataset/
*.zip

# 允许代码
!*.py
!*.json
!*.md
```

## 🖥️ SSH 终端

右键点击服务器 → "打开 SSH 终端"

现在你可以在 VS Code 中直接运行：
```bash
python train.py
tensorboard --logdir=./logs
```

## 🐛 常见问题

### Q: AI 说找不到文件？
A: 先手动打开那个文件，触发下载，然后 AI 就能访问了。

### Q: 扫描太慢了？
A: 点击"取消"，然后配置 `.svcignore` 忽略大文件夹。

### Q: 文件没有自动上传？
A: 检查 Output 面板 → "SVC Extension" 查看日志。

### Q: 如何删除服务器？
A: 左侧服务器列表 → 右键 → 删除
